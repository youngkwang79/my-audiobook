# Project Custom Rules

## ✍️ 블로그 통합 포스팅 생성 및 업로드 규칙 (Platform Multi-Poster Rule)

유저가 새로운 원고 또는 키워드로 글 업로드 패키지를 요청할 경우, 반드시 다음 상세 기준을 만족하여 수행한다.

1. **포커스 키워드**:
   * 유저가 제시한 [포커스 키워드]를 기반으로 SEO 글쓰기를 진행한다.

2. **📚 무림북용 (Supabase DB)**:
   * **이미지**: 2:3 비율 북커버 썸네일 AI 이미지 생성.
   * **텍스트**: 제목 글씨가 크게 오버레이된 마크다운 텍스트 생성.
   * **예약 발행**: 업로드 시점 기준 정확히 1시간 뒤 자동 예약 발행 설정 (`status = "공개예정"`, `created_at = 현재+1시간`).
   * **SEO 보강**: 본문 최하단에 자연스러운 내부 링크(부자 습관 칼럼 등), 외부 신뢰 링크(위키백과 등) 및 해시태그 강제 주입.

3. **🔵 워드프레스용 (WP REST API - DRAFT 등록)**:
   * **Rank Math SEO 90점 기준 충족**:
     * 글자 수 1,000자 이상으로 문맥 확장.
     * 키워드 밀도 보정 (본문 내 포커스 키워드 적절히 분산 배치).
     * 내부 링크(부자 습관 칼럼) 및 외부 신뢰 링크 포함.
     * 본문 이미지에 Alt 대체 텍스트(Alt Text) 설정 필수.
   * **메타 설정**:
     * 글 메뉴에 관련 태그 생성 및 매핑.
     * 관련 카테고리 3개 중복 체크 선택 (`AI/자동화`, `IT/업무`, `경제/산업` 등).
     * 제목 규칙: 포커스 키워드를 맨 앞에 배치하고, 숫자를 포함하여 공백 포함 20자 내외로 짓는다 (예: '전기세 절약 3대 비법...').
   * **미디어 연동**:
     * 16:9 규격의 대표 이미지(Featured)와 16:9 본문 중간 이미지를 워드프레스 미디어 서버에 자동 업로드하고 스니펫 메타데이터 설정하여 임시글(Draft)로 포스팅한다.

4. **🟢 구글 블로거용 (Blogger API - DRAFT 등록)**:
   * 중복 문서 페널티(저품질 블로그 낙인)를 피하기 위해, 동일한 주제를 완전히 다른 시각(정보 전달/테크니컬 소비전력 분석 위주)의 원고로 변환하여 임시 저장글로 자동 업로드.

5. **📂 티스토리 & 네이버 블로그용 (텍스트 아티팩트 제공)**:
   * **티스토리**: 마크다운 서식 전용 카카오/스마트홈 타깃 원고.
   * **네이버**: 줄글이 짧고 이모티콘과 공감이 풍부한 친근한 에세이형 모바일 가독성 원고.
